# Baza грузоперевозок

## Privacy policy

### Data collected by the app

Персональные данные, разрешённые к обработке в рамках настоящей Политики конфиденциальности, предоставляются Пользователем, и включают в себя следующую информацию:
- фамилию, имя, отчество Пользователя;
- контактный телефон Пользователя;
- адрес электронной почты (e-mail);
- ИИН/БИН пользователя;
- Название компаний;
- данные журналов и информацию об устройстве, когда был выполнен вход в Мобильное приложение;
- информацию о точном или приблизительном местонахождении, определенном с помощью таких данных, как ваш IP-адрес или GPS мобильного устройства;
- взаимодействие в Мобильном приложений.
2.3. Мобильное приложение защищает Данные, которые автоматически передаются в процессе просмотра рекламных блоков и при посещении страниц, на которых установлен статистический скрипт системы ("пиксель"):
• IP адрес;
• информация о браузере (или иной программе, которая осуществляет доступ к показу рекламы);
• время доступа;
• адрес страницы, на которой расположен рекламный блок;
• реферер (адрес предыдущей страницы).

### Explanation of permissions requested in the app

The list of permissions required by the app:

<br/>

| Permission | Why it is required |
| :---: | --- |
| `android.permission.INTERNET` | Allows the app to access the internet, which is essential for features that require online data, such as fetching updates, synchronizing with online servers, or accessing online resources. |
| `android.permission.ACCESS_NETWORK_STATE` | Enables the app to check the state of network connectivity (e.g., whether there is an active internet connection) and adjust its behavior accordingly, ensuring smooth operation and efficient data usage. |
| `android.permission.WAKE_LOCK` | Prevents the device from going into sleep mode while the app is performing a critical task, ensuring that the task is completed without interruption. |
| `android.permission.ACCESS_BACKGROUND_LOCATION` | Allows the app to access location information even when it is running in the background. This is crucial for providing location-based services and ensuring accurate location data is collected, even when the app is not actively in use. |
| `android.permission.ACCESS_COARSE_LOCATION` | Allows the app to access approximate location information, which is less accurate but consumes less power. It is used for features that do not require precise location data. |
| `android.permission.ACCESS_FINE_LOCATION` | Allows the app to access precise location information using GPS and network location services. This is essential for features that require accurate location data, such as navigation or location-based services. |
| `android.permission.FOREGROUND_SERVICE` | Enables the app to create and run a foreground service, which is a service that the user is actively aware of and which is essential for tasks that need to continue running even when the app is not in the foreground. |
| `android.hardware.location.network` | Indicates that the app can use network-based location services. Marked as optional to ensure the app can still function on devices without this feature. |
| `android.hardware.location.gps` | Indicates that the app can use GPS-based location services. Marked as optional to ensure the app can still function on devices without this feature. |
| `android.permission.CAMERA` | Allows the app to access the device's camera, which is necessary for features that involve capturing photos or videos. |
| `android.permission.RECORD_AUDIO` | Allows the app to access the device's microphone to record audio. This is essential for features that require audio input, such as voice recording or communication.|
| `android.permission.WRITE_EXTERNAL_STORAGE` | Allows the app to write data to the device's external storage, which is necessary for saving files, such as images, videos, or documents.|
| `android.permission.READ_EXTERNAL_STORAGE` | Allows the app to read data from the device's external storage, which is essential for accessing files stored on the device.|
| `android.permission.VIBRATE` | Enables the app to trigger the device's vibration, which can be used for notifications or alerts.|
| `android.permission.RECEIVE_BOOT_COMPLETED` | Allows the app to automatically start or perform certain tasks when the device is rebooted, ensuring continuity of operations, such as setting alarms or restoring notifications.|
